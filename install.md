# Cryptova Installation and Setup Guide

Cryptova is an AI-powered cryptocurrency trading assistant system.

The project consists of a React frontend, FastAPI backend, AI inference server, and MySQL database. Users can register, log in, configure trading strategies, and validate an AI-driven trading workflow through a paper trading environment using LONG / SHORT / HOLD signals generated by the AI model.

---

## 1. Environment Requirements

This project is designed to run in a local development environment.

| Category | Technology |
|----------|------------|
| Frontend | React, Vite, JavaScript |
| Backend | Python, FastAPI, SQLAlchemy |
| AI Server | Python, FastAPI |
| Database | MySQL |
| External APIs | Bybit Public API, Gemini API |
| Package Managers | npm, pip |

Recommended environment:

```text
Python 3.10+
Node.js 18+
npm
MySQL 8.x
Git
VS Code or PowerShell
```

## 4. Database Setup

Connect to MySQL and create the database used by the project:

```sql
CREATE DATABASE cryptova_db;
```

If the database already exists, this step can be skipped.

The backend uses SQLAlchemy's `Base.metadata.create_all()` to automatically create required tables when the server starts.

Main tables created by the system:

```text
users
strategy_settings
ai_signals
orders
trading_positions
trading_runs
api_keys
backtest_results
```

---

## 5. Backend Environment Variables

Create a `.env` file inside the `cryptova-back` directory.

```env
APP_NAME=Cryptova
APP_ENV=local

DATABASE_URL=mysql+pymysql://root:password@localhost:3306/cryptova_db

JWT_SECRET_KEY=change-this-secret-key
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=60

AUTO_TRADING_INTERVAL_MINUTES=60

GEMINI_API_KEY=YOUR_GEMINI_API_KEY
BYBIT_PUBLIC_BASE=https://api.bybit.com
```

### Notes

- Replace the MySQL password in `DATABASE_URL` with your own credentials.
- If `GEMINI_API_KEY` is not provided, fallback explanations will be generated instead of Gemini-generated explanations.
- `AUTO_TRADING_INTERVAL_MINUTES` controls the automated trading interval.
- For testing, 1 minute is recommended; for normal operation, 60 minutes is recommended.

## 6. Backend Installation and Execution

Move to the backend directory:

```bash
cd cryptova-back
```

Create a virtual environment:

```bash
python -m venv venv
```

Activate the environment.

### Windows

```bash
venv\Scripts\activate
```

### Linux / macOS

```bash
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

If `requirements.txt` is unavailable, install the core packages manually:

```bash
pip install fastapi uvicorn sqlalchemy pymysql python-dotenv python-jose passlib[bcrypt] apscheduler httpx requests google-generativeai
```

Start the backend server:

```bash
uvicorn app.main:app --reload
```

Backend URL:

```text
http://localhost:8000
```

Swagger documentation:

```text
http://localhost:8000/docs
```

## 11. Common Issues and Solutions

### 11.1 `/auth/me 401 Unauthorized`

#### Cause

Missing or expired authentication token.

#### Solution

1. Log out and log in again.
2. Verify `access_token` in browser localStorage.
3. Check login flow if the token is not being stored.

---

### 11.2 `/exchange/balance 404 Not Found`

#### Cause

No Bybit API key has been registered.

#### Solution

Cryptova primarily operates in Paper Trading mode. Exchange balance errors do not affect paper trading functionality.

---

### 11.3 `Bybit API key is not registered`

#### Cause

Exchange API credentials have not been stored.

#### Solution

Continue testing with Paper Trading mode. No real trades are required.

---

### 11.4 `Too many visits. Exceeded the API Rate Limit`

#### Cause

The Bybit Public API has been called too frequently.

#### Solution

1. Avoid excessive page refreshes.
2. Avoid rapid timeframe switching.
3. Consider backend caching or database persistence improvements.

---

### 11.5 `GEMINI_EXPLANATION_ERROR`

#### Cause

Gemini quota limits, invalid API key, incorrect model configuration, or timeout issues.

#### Solution

1. Verify `GEMINI_API_KEY` configuration.
2. Check Gemini quota limits.
3. The system automatically falls back to a rule-based explanation if Gemini fails.

---

### 11.6 AI Server Connection Failure

#### Cause

The AI server is not running or the configured port is incorrect.

#### Solution

Restart the AI server:

```bash
cd cryptova-ai
uvicorn app:app --host 0.0.0.0 --port 8001 --reload
```

Verify the backend is using:

```text
http://localhost:8001
```
